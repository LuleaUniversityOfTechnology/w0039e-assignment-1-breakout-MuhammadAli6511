Assignment 1.2: Score system in Breakout

  • Due 21 Nov 2024 by 23:59
  • Points 0.5
  • Submitting a website url
  • Available after 11 Nov 2024 at 0:00

This assignment has the following objectives

  1. More practice with functions
  2. Practice using pointers
  3. Practice using File I/O

The assignment builds upon the previous.

Use the same repository as previously. Post the link to the repository as your submission to this assignment.

Add a high score system

1. Create a stack array that contains the top 5 high scores (unsigned integers).
2. Draw the high score array to the screens bottom right using `Play::DrawDebugText`.
3. Add a variable that represents the current player score.
4. Draw the current player score to the bottom left of the screen.
5. When the ball breaks a brick, increase the current player score by 1.
6. When the ball exits the bottom of the screen, store the current score in the high score array at the appropriate place. Make sure you shift all the previous scores if necessary. Then remove the ball and all remaining bricks, reset the scene and the current score and spawn a new ball.

Add file input and output

1. Replace the stack array with a dynamically allocated array of unsigned integers. The size will be arbitrary, but start out by just creating 5 elements.
2. When exiting the game, write the highscore to a text file using the `fstream` library.
3. When starting the game, read the highscore from the text file, and store the values in the highscore array.
4. Make sure that the highscore array in the game is big enough to fit all the scores that are read from the highscore text file, so that if you were to insert 6 highscores in the file, the program will read and store all 6 of them in the application.

    Hint: the size of the highscore array could be equal to the amount of lines in the text document.

5. Make sure you delete any allocated memory before exiting the application.

    If you do not delete allocated memory properly, there will be a memory leak, which is not permitted. You can check whether there is a memory leak happened in the 'Output' window.
